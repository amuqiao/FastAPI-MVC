# Mermaid 作图提示词

这份文档用于沉淀 `mermaid-analysis-visualizer` 的常用调用方式。  
目标不是重复 skill 规范，而是给出几份你可以直接复制使用的 Prompt。

## 一、先记住最短公式

最短正确写法是：

```md
使用 $mermaid-analysis-visualizer + 具体任务
```

例如：

```md
使用 $mermaid-analysis-visualizer 画一张系统架构图。
```

这句话里：
- `画一张系统架构图` 是任务
- `$mermaid-analysis-visualizer` 是启用的能力

skill 不替代你的提问，skill 是增强你的提问。

## 二、最短通用版

```md
使用 $mermaid-analysis-visualizer 输出 Mermaid 图。

要求：
- 先用 2 到 4 句话说明这张图回答什么问题
- 再输出 Mermaid 代码
- 最后补一段图解说明
- 如果信息不足，请明确指出缺口，不要编造不存在的组件关系
- 严格遵循 skill 内对应 reference 的图型规范、配色风格、分层方式、NOTE 注记和 linkStyle 写法
- 不要自创一套新的视觉风格
```

## 三、系统架构图提示词

```md
使用 $mermaid-analysis-visualizer 输出一张系统架构图。

背景：
这是一个 {你的系统名称}。

目标：
我想看清这个系统有哪些核心层、核心服务，以及它们之间如何协作。

要求：
- 使用 A 类“系统认知层”规范
- 输出系统架构图，不要输出时序图或状态机图
- 保持分层清晰，优先展示：客户端层、接入层、业务服务层、基础设施层、外部服务层
- 严格沿用 reference 中的视觉语法：classDef、subgraph、NOTE、边索引注释、linkStyle
- 优先替换领域节点，不要更换整套图面风格
- 先简短分析，再给 Mermaid 代码，最后给图解说明
- 如果信息不足，请明确指出缺口，不要编造组件关系
```

## 四、端到端流程图提示词

```md
使用 $mermaid-analysis-visualizer 输出一张端到端流程图。

背景：
主题是 {你的主题，例如：AI 漫剧生成流程 / 用户下单流程 / RAG 问答流程}。

目标：
我想看清一个请求或一条业务主线是如何同步流转的。

要求：
- 使用 A 类“系统认知层”中的端到端流程图规范
- 输出 flowchart LR
- 只画主流程，不展开异步回调和状态机细节
- 沿用 reference 中的配色风格、分组方式、NOTE 注记和 linkStyle 风格
- 先说明这张图回答什么问题
- 再输出 Mermaid 代码
- 最后补图解说明
- 如果信息不足，不要硬补不存在的步骤
```

## 五、信息不完整时的安全版提示词

```md
使用 $mermaid-analysis-visualizer 输出一张 {系统架构图 / 流程图}。

当前已知信息：
- 主题：{例如 AI 漫剧系统}
- 已知模块：{模块1}、{模块2}、{模块3}
- 已知外部依赖：{依赖1}、{依赖2}
- 已知主链路：{一句话说明}

要求：
- 基于已知信息只输出高确定性的内容
- 缺失部分请明确写出，不要推测成事实
- 严格遵循对应 reference 的图型和视觉风格
- 先简短分析，再给 Mermaid 代码，再给图解说明
```
