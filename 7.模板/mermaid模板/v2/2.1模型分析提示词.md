## 模型分析
分析深度学习模型 {模型名称}，请编写一份专业的技术分析文档，涵盖以下内容：
1. **模型定位**：一句话说清楚模型解决什么问题、属于哪个研究方向，核心创新点是什么
2. **整体架构**：按「功能模块 → 子模块 → 关键算子」三层拆解，说明各层的职责边界和模块间连接方式（串行 / 并行 / 跨模块特征复用）。**必须使用 ASCII 树形结构**（`├──` / `└──` / `│`）输出，每行末尾用空格对齐后附简短功能说明，禁止使用表格。
3. **数据直觉**：以 1 条具体样例为主线，用人类可读的形式完整展示数据在各阶段的形态变化：原始输入（真实文字/图像/序列内容）→ 预处理后（tokenization 序列、patch 切分、归一化数值等）→ 关键中间表示（如 embedding 向量含义、注意力分布、特征图语义等，不只是维度，要说明「这一步在表达什么」）→ 模型输出（原始输出形式）→ 后处理结果（最终可用的预测/生成内容及对应标签），帮助直观理解「数据在模型里经历了什么」
4. **核心数据流**（重点）：数据从输入到输出的完整流转路径，标注每个关键节点的张量维度变化
5. **关键组件**：选取最核心的 `1~3` 个组件，先用 `1~2` 句直觉语言说清楚「这个机制本质上在做什么」，再深入分析其内部计算原理（含数学公式）
6. **训练策略**：损失函数设计、优化器与学习率调度、关键训练技巧
7. **评估指标与性能对比**：主要评估指标的含义与选用原因；在核心 Benchmark 上与代表性基线的对比结果；关键消融实验中各组件的贡献；参数量/FLOPs/推理延迟等效率指标
8. **推理与部署**：推理阶段与训练阶段的差异（如 BN 模式、Dropout 关闭等）；输出后处理流程（NMS、beam search、解码等，视任务而定）；常见的部署优化手段（量化、蒸馏、ONNX 导出等，如有）
9. **FAQ 12个以上**：每题详细解答，覆盖基本原理、设计决策、实现细节、性能优化四类问题

质量要求：文档结构清晰，分章节组织；重点分析「为什么这样设计」而非仅描述「结构是什么」；对每个核心机制遵循「直觉 → 原理 → 公式」的递进顺序，避免直接抛出公式；数学公式使用 $...$ 和 $$...$$ 语法
格式要求：Markdown，使用 Mermaid 图辅助说明。
**必须包含以下图**：
- 模型整体架构图 → 建立模块认知
- 前向传播/张量流图 → 追踪数据变换
- 模块内部结构图 → 深入核心组件
- 训练流程图 → 理解优化过程
- 数据处理流水线图 → 理解数据输入

**可选补充图（遇到以下情况时主动添加，无需逐一覆盖）**：
- 注意力机制 / 位置编码 / 门控结构等非线性机制 → 用图展示信息流与权重分配
- 多分支 / 跨层连接 / 特征复用路径 → 文字难以清晰表达时
- 推理阶段流程差异（如解码策略、NMS、后处理链路） → 与训练流程有显著差异时
- 消融实验中各组件的贡献对比 → 帮助理解设计决策的必要性
- 部署优化链路（量化 / 蒸馏 / 导出流程） → 有具体步骤时
- 其他任何「一张图能替代三段文字」的地方
