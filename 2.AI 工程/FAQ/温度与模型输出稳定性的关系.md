# Q1：Temperature 与模型输出稳定性

---

**Q：为什么温度（Temperature）越低，模型输出越稳定？温度越高，输出越不稳定？**

**A：**

本质原因在于 **Temperature 控制了 Softmax 输出的概率分布形状**，进而决定采样的随机程度。

### 数学机制

Softmax with temperature 的公式为：

$$P(x_i) = \frac{e^{z_i / T}}{\sum_j e^{z_j / T}}$$

其中 $z_i$ 为 logits，$T$ 为温度系数。

---

### 低温（T < 1）→ 输出稳定

- $T < 1$ 会**放大** logits 之间的相对差距；
- Softmax 后概率分布**极度尖锐（sharp）**：头部 token 概率趋近 1，其余趋近 0；
- 采样几乎总是选中概率最大的 token，路径高度确定；
- 分布熵（entropy）极低，输出**收敛、保守、不跑偏**。

> 一句话：**低温 → 分布锐化 → 熵低 → 采样确定性强 → 输出稳定**

---

### 高温（T > 1）→ 输出不稳定

- $T > 1$ 会**压缩** logits 之间的相对差距；
- Softmax 后概率分布**趋于均匀（smooth）**：高分 token 概率被压低，低分 token 概率被抬高；
- 原本几乎不会被选中的 token 也有较大概率被采样，路径高度随机；
- 分布熵极高，输出**发散、易跑题、容易产生逻辑混乱**。

> 一句话：**高温 → 分布平坦化 → 熵高 → 采样随机性强 → 输出不稳定**

---

### 对比总结

| 维度 | 低温（T ↓） | 高温（T ↑） |
|------|------------|------------|
| 概率分布形状 | 尖锐（sharp） | 平坦（smooth） |
| 分布熵 | 低 | 高 |
| 采样确定性 | 强 | 弱 |
| 输出表现 | 稳定、保守、收敛 | 随机、发散、易出错 |
| 典型应用场景 | 代码生成、事实问答 | 创意写作、头脑风暴 |

---

### 追问：T=0 和 T→∞ 分别是什么极端情况？

- **T → 0**：退化为 **greedy decoding（贪心解码）**，每次严格选概率最大的 token，输出完全确定；
- **T → ∞**：概率分布趋于**完全均匀分布**，每个 token 被选中的概率几乎相等，输出完全随机无意义。

# Q2：Logits 是什么？

## 一句话定义

**Logits 是模型最后一层线性层的原始输出值，是未经归一化的"原始分数"。**

---

## 在语言模型中的位置

```
输入 token
    ↓
Transformer 各层处理
    ↓
最后一层隐藏状态 hidden state（维度：d_model）
    ↓
线性层（Linear / LM Head）映射到词表大小
    ↓
输出 logits（维度：vocab_size，例如 32000）   ← 就是这里
    ↓
Softmax → 概率分布
    ↓
采样 → 生成下一个 token
```

---

## 直观理解

假设词表有 3 个词：`["猫", "狗", "鱼"]`，模型输出的 logits 可能是：

| Token | Logits（原始分数） | Softmax 后概率 |
|-------|------------------|---------------|
| 猫 | 3.2 | 0.70 |
| 狗 | 1.8 | 0.18 |
| 鱼 | 1.5 | 0.12 |

- Logits **没有上下界**，可以是任意实数（正、负、零都行）；
- 数值越大，表示模型"认为"该 token 越应该出现；
- 经过 **Softmax** 后才变成合法概率（0~1，且总和为 1）。

---

## 为什么叫"Logits"？

来源于统计学中的 **log-odds（对数几率）**：

$$\text{logit}(p) = \log\frac{p}{1-p}$$

深度学习中沿用了这个名字，泛指**Softmax 之前的原始线性输出**，是一种约定俗成的叫法。

---

## 与 Temperature 的关系（接上文）

Temperature 就是直接作用在 logits 上的：

$$P(x_i) = \frac{e^{z_i / T}}{\sum_j e^{z_j / T}}$$

- $z_i$ 就是 logits；
- 先对 logits 除以 $T$，再送入 Softmax；
- 所以 Temperature 本质上是在**缩放 logits 的差距**，进而改变概率分布的形状。

---

### 一句话记忆

**Logits = Softmax 之前的原始打分，数值越大代表模型越倾向选这个 token。**
