# 知识库提示词模板库

> 定位：这是面向日常使用的 prompt 模板集合，用于配合 [知识库构建与维护规范.md](/Users/admin/Downloads/Code/empty/skill开发规范/知识库构建与维护规范.md) 和 [知识库极简操作清单.md](/Users/admin/Downloads/Code/empty/skill开发规范/知识库极简操作清单.md) 使用。

---

## 1. 使用原则

使用这些 prompt 时，先记住四条规则：

1. 先判断场景，再选模板。
2. 非必要不要让 AI 重写整套知识库。
3. 默认要求 AI 只更新最小必要文件。
4. 不确定内容先进入待核验区，不直接写成正式结论。

---

## 2. 新建主题

适用情况：

- 这是一个全新的主题
- 还没有知识库目录
- 需要从原始材料出发搭建结构

模板：

```text
请围绕主题《XXX》构建一套适合长期维护的知识库。

背景材料如下：
<<<在这里粘贴原始材料或摘要>>>

任务目标：
1. 识别该主题的基础概念、核心原理、工程实践、平台差异、待核验项
2. 设计适合该主题规模的目录结构
3. 生成 README、主文档，以及必要的子文档
4. 识别重复内容并合并
5. 对冲突内容单独标注，不强行混为一谈

要求：
- README 只负责导航、阅读顺序、状态和维护规则
- 主文档只负责总纲、稳定结论、子主题索引
- 子文档按单一主题展开
- 如果主题规模不大，不要过度拆分
- 命名统一，适合长期维护

输出要求：
- 先给出建议目录结构
- 再给出各文件职责说明
- 最后输出各文档正文
```

## 3. 基于现有目录补充新材料

适用情况：

- 主题已经存在
- 你拿到了新增资料
- 你不希望 AI 重写整库

模板：

```text
请基于新增材料更新现有知识库，不要重写整套文档。

当前目录结构如下：
<<<在这里粘贴当前目录结构>>>

现有文档职责如下：
<<<在这里粘贴 README 或主文档中的职责说明>>>

新增材料如下：
<<<在这里粘贴新增材料>>>

更新目标：
1. 判断新增内容属于哪个现有文件
2. 只更新最小必要文件
3. 保持原有目录结构和命名稳定
4. 将稳定结论写入正文
5. 将不确定、版本相关、来源冲突内容写入待核验区或平台差异区
6. 不要把相同内容重复写入 README、主文档和子文档

输出要求：
- 先说明本次应该更新哪些文件，为什么
- 再输出更新后的具体内容
- 如果某些文件无需修改，明确说明“不需要修改”
```

## 4. 只修订一个已有文档

适用情况：

- 你已经知道问题出在哪个文件
- 只需要局部修订
- 不想让 AI 改动别的文件

模板：

```text
请只修订以下文档，不要改动知识库中的其他文件。

目标文件：
《XXX》

当前文档内容：
<<<在这里粘贴目标文件内容>>>

新增信息或修订依据：
<<<在这里粘贴新增内容>>>

修订要求：
1. 保留原有结构和命名
2. 只做必要修订
3. 不新增无关章节
4. 如有不确定内容，单独标注为待核验
5. 不要重写整篇文档的表达风格

输出要求：
- 先说明修订点
- 再输出修订后的完整文档
```

## 5. 把待核验项转为正式结论

适用情况：

- 某条待核验内容已经有了更稳定依据
- 你希望把它写入正式文档

模板：

```text
请判断以下待核验内容是否可以转为正式结论，并更新知识库。

当前待核验内容：
<<<在这里粘贴待核验项>>>

新增依据：
<<<在这里粘贴依据、材料或说明>>>

相关文档：
<<<在这里粘贴主文档或对应子文档内容>>>

任务要求：
1. 判断该内容是否已足够稳定，可以进入正式结论
2. 如果可以，说明应写入哪个文档
3. 如果不可以，说明还缺什么
4. 不要把依据不足的内容写成正式结论

输出要求：
- 先给出判断结果
- 再给出需要更新的文件
- 最后输出更新后的内容
```

## 6. 检查知识库结构是否失控

适用情况：

- 文档越来越多
- 你怀疑出现了重复、混写、越权扩张

模板：

```text
请检查当前知识库的结构是否适合长期维护。

当前目录结构如下：
<<<在这里粘贴目录结构>>>

关键文档如下：
<<<在这里粘贴 README、主文档或相关子文档内容>>>

检查目标：
1. 是否存在 README 与主文档职责重叠
2. 是否存在子文档之间内容重复
3. 是否存在应拆分但尚未拆分的主题
4. 是否存在不应写死却被写成正式结论的内容
5. 是否存在可以转正的待核验项

输出要求：
- 先给出结构诊断
- 再给出最小修改建议
- 非必要不要建议重构整个目录
```

## 7. 合并多篇零散笔记为知识库

适用情况：

- 你有多篇笔记
- 内容有重复、冲突、顺序混乱
- 想整理成正式知识库

模板：

```text
请将我提供的多篇笔记整理为一套适合长期维护的知识库。

原始笔记如下：
<<<在这里粘贴多篇笔记>>>

任务要求：
1. 识别重复内容并合并
2. 按合理顺序重排章节
3. 区分基础概念、核心原理、工程实践、扩展讨论
4. 对冲突内容进行标注，不要强行混为一谈
5. 输出适合长期维护的知识库目录结构
6. 生成 README、主文档和必要的子文档

输出要求：
- 先给出目录结构
- 再给出正式文档
- 再说明重复内容如何收敛
- 最后列出仍待确认的问题
```

## 8. 只做目录设计，不展开正文

适用情况：

- 你还不准备写内容
- 只想先搭骨架

模板：

```text
请围绕主题《XXX》先设计一套适合长期维护的知识库目录结构，不要展开正文。

背景材料如下：
<<<在这里粘贴主题说明或材料摘要>>>

任务要求：
1. 判断该主题适合小型、中型还是大型知识库
2. 给出推荐目录结构
3. 说明 README、主文档、子文档的职责边界
4. 指出哪些主题应该独立成子文档
5. 指出哪些内容暂时不需要单独建目录

输出要求：
- 只输出目录结构与职责说明
- 不要写正文草稿
```

## 9. 只做内容分层，不改目录

适用情况：

- 目录已经确定
- 你只想让 AI 判断内容应分别归到哪里

模板：

```text
请在不修改现有目录结构的前提下，对以下材料进行内容分层。

当前目录结构如下：
<<<在这里粘贴目录结构>>>

材料如下：
<<<在这里粘贴新材料>>>

任务要求：
1. 判断每部分内容应归入哪个现有文档
2. 区分稳定结论、平台差异、待核验内容
3. 不新增目录
4. 不重写已有文档

输出要求：
- 先给出内容归类结果
- 再说明每个文件应新增什么
- 不直接输出整库重写结果
```

适用情况：

- 已有知识库能看，但风格不统一
- 文件命名和章节命名混乱

模板：

```text
请对当前知识库进行“结构不变、内容风格统一”的规范化整理。

当前目录结构：
<<<在这里粘贴目录结构>>>

当前文档内容：
<<<在这里粘贴相关文档>>>

任务要求：
1. 保持现有目录结构不变
2. 保持现有文件名尽量不变
3. 统一标题层级、命名方式和表达口径
4. 清理重复表达
5. 不要扩大范围，不要新增无关文档

输出要求：
- 先说明需要调整的文件
- 再给出调整后的内容
```

## 11. 使用建议

### 11.1 最常用的选择方式

如果你不想记太多，只按这个规则选：

- 新主题：用“新建主题”
- 新材料：用“基于现有目录补充新材料”
- 单文件修订：用“只修订一个已有文档”
- 待核验转正：用“把待核验项转为正式结论”
- 结构体检：用“检查知识库结构是否失控”

### 11.2 每次使用前先补三类上下文

为了让 AI 输出更稳定，建议你每次都补这三类信息：

1. 当前目录结构
2. 当前目标文件内容
3. 本次新增材料或修订依据

### 11.3 最容易出问题的地方

最常见错误是：

- 没有给现有目录结构
- 没有说明“不要重写整套文档”
- 没有说明该次更新的范围
- 没有要求把不确定内容单独标注

## 12. 最小记忆版

如果你只记 5 句话，就记这 5 句：

1. 新主题先建结构，再写正文。
2. 老主题新增内容时，只更新最小必要文件。
3. README 只做导航，主文档只做总纲。
4. 平台差异和待核验项不要写进稳定结论。
5. 每次给 AI 下任务时，都要明确“保持目录稳定，不要重写整库”。
