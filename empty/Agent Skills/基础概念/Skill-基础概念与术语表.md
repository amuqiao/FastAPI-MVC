# Skill 基础概念与术语表

## 1. Skill 是什么

Skill 是一类任务的可复用工作流包。它的目的不是替代模型，而是把反复出现、边界相对稳定的做法沉淀成可复用说明。

它通常适合这类任务：

- 同一套流程重复出现
- 流程稳定，但仍需要模型理解上下文
- 希望跨会话复用同一套做法

不适合立刻写成 Skill 的情况：

- 一次性问题
- 纯探索问题
- 边界尚未理清的问题

## 2. 与相近概念的区别

| 概念 | 核心作用 | 与 Skill 的区别 |
|------|----------|----------------|
| 普通提示词 | 一次性交代 | 不强调复用、结构化和延迟加载 |
| Skill | 流程封装 | 关注触发边界、步骤结构和复用 |
| MCP | 能力接入 | 负责工具和外部上下文，不负责定义任务流程 |
| Plugin | 分发单元 | 更偏安装和分享，不等于 Skill 本体 |
| Cursor Rule | 规则体系 | 与 `SKILL.md` 体系有关联，但不等价 |

## 3. Skill 包中的常见元素

| 元素 | 作用 |
|------|------|
| `SKILL.md` | 核心入口，定义触发边界、步骤、输出和校验 |
| `references/` | 给模型按需读取的长资料 |
| `scripts/` | 稳定、机械、确定性的执行步骤 |
| `assets/` | 产物直接会用到的模板和样例 |
| `agents/openai.yaml` | 平台展示或平台侧元数据 |

## 4. 最重要的初学者判断

理解 Skill 时，先记住四句话：

1. Skill 不是更长的 prompt，而是可复用工作流。
2. `SKILL.md` 是核心入口。
3. `description` 决定自动匹配质量。
4. Skill 管流程，MCP 管能力接入。
