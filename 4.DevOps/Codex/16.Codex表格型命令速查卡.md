# Codex 表格型命令速查卡

> **文档职责**：把 Codex 常用命令压缩成一张表，适合初学者快速扫一眼。  
> **适用场景**：你想要的是命令 cheatsheet，而不是长文档。  
> **说明**：命令基于你本机当前 `codex-cli 0.120.0` 的帮助输出整理。

## 1. 最常用：先看这张

| 命令 | 作用 | 什么时候用 |
|---|---|---|
| `codex` | 启动交互模式 | 日常使用；也是“新建新会话”的最直接方式 |
| `codex --help` | 看全部帮助 | 忘记命令时 |
| `codex --version` | 看版本 | 排错、确认是否安装成功 |
| `codex --cd <目录>` | 从指定目录启动 | 只想看某个子模块 |
| `codex --model <模型>` | 启动时指定模型 | 你明确知道要用哪个模型 |
| `codex --full-auto` | 低摩擦自动执行模式 | 日常本地开发 |
| `codex resume --last` | 继续上次会话 | 昨天做到一半 |
| `codex fork --last` | 从上次会话分叉 | 想试第二种方案 |

## 3. 会话管理命令

| 命令 | 作用 | 备注 |
|---|---|---|
| `codex` | 新建新会话 | 终端里最直接的“重新开始”；等价于放弃当前上下文重新开一条 |
| `codex resume` | 从列表里选旧会话继续 | 不确定是哪一个时；适合回到旧任务 |
| `codex resume --last` | 继续最近一次会话 | 最常用；适合昨天做到一半；如果最近会话已经跑偏，别硬接，直接 `codex` 或 `/new` |
| `codex resume <SESSION_ID>` | 指定会话继续 | 已知 session id 时；精确回到目标会话 |
| `codex fork` | 从列表里选旧会话分叉 | 保留原会话 |
| `codex fork --last` | 从最近一次会话分叉 | 最常用；适合保留原思路同时试新方案；如果不需要保留旧路线，直接开新会话更干净 |
| `codex fork <SESSION_ID>` | 指定会话分叉 | 适合试新方案 |

## 7. 会话内斜杠命令

这部分是在 Codex 交互界面里输入，不是在系统终端里执行。

| 命令 | 作用 | 备注 |
|---|---|---|
| `/new` | 新建新会话 | 最适合“我要重开，不要旧上下文”；它不是恢复，也不是压缩 |
| `/model` | 看或切模型 | 高频 |
| `/permissions` | 看或切权限 | 高频 |
| `/mention path/to/file` | 挂文件到上下文 | 高频 |
| `/compact` | 压缩长会话 | 很实用；当上下文很长、回复开始变散时优先用；如果你已经换问题，直接 `/new` 更合适 |
| `/resume` | 继续旧会话 | 会话内也可用；适合明确要接回旧线程；如果只是想摆脱旧上下文，不要用它，直接 `/new` |
| `/fork` | 分叉当前思路 | 保留原线程 |
| `/init` | 生成 `AGENTS.md` 模板 | 仓库初始化时用 |

> 斜杠命令可能随版本变化。最稳的做法是进入 Codex 后输入 `/` 看当前界面提示。

### `/new`、`/resume`、`/compact` 怎么选

| 命令 | 什么时候优先用 | 一句话判断 |
|---|---|---|
| `/new` | 你不想继承当前上下文，准备重新开始一个问题 | **我要重开，不要旧上下文** |
| `/resume` | 你明确要接回之前的某条旧会话 | **我要续上旧线程** |
| `/compact` | 你还想留在当前会话，但上下文已经太长 | **我还在这条线上，只是想压缩上下文** |

补充理解：

- 如果你只是想“别让旧子会话继续影响我”，通常优先用 **`/new`**
- 如果你是“昨天做到一半，今天继续”，优先用 **`/resume`**
- 如果你觉得“还是这个问题，但上下文太长、回复开始发散”，优先用 **`/compact`**
