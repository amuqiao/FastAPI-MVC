## 一、输入框：换行与发送 ⌨️

**日常最常用就这一节**，别的忘了再来翻。

### 1.1 换行（推荐）

| 操作 | 快捷键 |
|------|--------|
| 换行（不发送） | `Shift + Enter` |
| 发送 | 单独按 `Enter` |

适用于 Codex、Cursor 及多数同类 AI 编辑器。

**多行示例（每段后按 Shift+Enter）：**

```
帮我写一个函数
功能是计算数组总和
支持整数和浮点数
```

### 1.2 粘贴长文本

- 直接粘贴会保留原有换行。
- 粘贴不会自动发送；确认无误后再按 `Enter` 发送。（放心贴，不会「手滑发出去」😅）

### 1.3 分段写指令（结构示例）

```
上下文：
@utils.py

需求：
1. 优化函数
2. 增加异常处理
3. 支持浮点数

注意：
保持函数名不变
```

### 1.4 Codex CLI（命令行）

终端里别跟 GUI 较劲，**反斜杠续行**搞定：

多行字符串用反斜杠续行：

```bash
codex "帮我写一个函数 \
功能是计算数组总和 \
支持整数和浮点数"
```

## 二、通用多行模板 📋

**复制 → 改 `xxx` 和 `@文件` → 分段用 Shift+Enter**，懒得排版就套这个。

### 模板 1：改代码 / 修 Bug

```
帮我修改下面这段代码，需求如下：

1. 修复逻辑错误
2. 增加参数校验和异常处理
3. 保持原有函数名和调用方式不变
4. 代码简洁，注释清晰

相关文件：
@文件名.py

代码片段：
粘贴你要改的代码
```

### 模板 2：新增功能 / 写函数

```
帮我实现一个功能，需求：

1. 功能描述：xxx
2. 输入参数：xxx
3. 返回结果：xxx
4. 约束条件：xxx
5. 编码风格：简洁、类型提示、注释清晰

参考文件：
@xxx.py

请直接输出完整可运行代码，并简单说明改动点。
```

### 模板 3：通读文件、只改局部

```
请先阅读 @文件名.py
重点关注第 xx 行 ~ xx 行的逻辑。

需求：
1. xxx
2. xxx
3. xxx

只修改关键逻辑，不要大面积重构，保持原有结构。
```

### 模板 4：项目级（多目录上下文）

```
我现在在开发一个 xxx 项目，技术栈：xxx

需要实现：xxx

已加载目录：
@src
@utils

请先梳理实现思路，再给出关键代码文件的完整实现。
```

### 模板 5：极简万能（想少打字用这个）

```
需求：
- xxx
- xxx
- xxx

参考：
@文件1
@文件2

代码：
粘贴代码片段
```
