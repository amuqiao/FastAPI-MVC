# Codex 新手 3 分钟上手卡片

> **文档职责**：只保留初学者第一天最需要的命令和顺序。  
> **适用场景**：你刚装好 Codex，只想知道“先敲什么、再敲什么”。  
> **使用方式**：从上往下照着做，不用一次记住全部。

## 1. 第一步：确认装好了

```bash
codex --version
```

如果能看到版本号，说明 CLI 已经可用。

---

## 2. 第二步：进入项目目录

```bash
cd your-project
```

如果你只想看某个子模块，后面也可以直接：

```bash
codex --cd services/payments
```

---

## 3. 第三步：启动 Codex

```bash
codex
```

这是最基础、最常用的命令。

---

## 4. 第四步：第一句话这样说

进入 Codex 后，先发这句：

```text
先阅读当前仓库，告诉我项目结构、启动方式和测试方式，先不要修改文件。
```

这一步的目的不是改代码，而是先让 Codex 理解项目。

---

## 5. 第五步：开始做真正任务

比如修 bug，可以直接说：

```text
修复“XXX”问题，在最小改动范围内处理，不要改无关模块，改完后运行相关验证。
```

比如只分析，可以说：

```text
分析“XXX”问题的根因和最小修复方案，先不要修改文件。
```

---

## 6. 会话里最常用的 5 个命令

### 6.1 看或切模型

```text
/model
```

### 6.2 看或切权限

```text
/permissions
```

### 6.3 挂关键文件

```text
/mention path/to/file
```

### 6.4 压缩长会话

```text
/compact
```

### 6.5 初始化协作规则

```text
/init
```

---

## 7. 终端里最常用的 5 个命令

```bash
codex
codex --help
codex --version
codex --cd <目录>
codex resume --last
```

如果你昨天做到一半，继续上次任务：

```bash
codex resume --last
```

如果你想从上次任务分一条新思路：

```bash
codex fork --last
```

---

## 8. 初学者最常犯的 3 个错误

### 8.1 一上来就让它盲改

不推荐：

```text
直接把这个项目改好。
```

更推荐：

```text
先阅读项目并分析问题，先不要修改文件。
```

### 8.2 没有限制改动范围

不推荐：

```text
修一下这个 bug。
```

更推荐：

```text
只修改订单创建链路和相关测试，不要改无关模块。
```

### 8.3 改完不要求验证

一定记得补一句：

```text
改完后运行相关验证并总结结果。
```

---

## 9. 你现在只要记住这 3 句话

```bash
codex
```

```text
先阅读当前仓库，告诉我项目结构、启动方式和测试方式，先不要修改文件。
```

```text
改完后运行相关验证并总结结果。
```

如果这 3 句你已经能熟练用，就已经能开始日常协作了。
