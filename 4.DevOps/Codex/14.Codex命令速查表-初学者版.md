# Codex 命令速查表（初学者版）

> **文档职责**：只讲 Codex 的高频命令，不展开长篇原理。  
> **适用场景**：你刚开始用 Codex，想知道“这个命令是干嘛的、什么时候用”。  
> **说明**：本文按你本机当前 `codex --help` 输出整理，适合日常速查。

## 1. 先记住最常用的 8 个

```bash
codex
codex --help
codex --version
codex --cd <目录>
codex --model <模型名>
codex --full-auto
codex resume
codex fork
```

如果你是初学者，先把这 8 个用熟，再看后面的细项。

---

## 2. 最基础：启动 Codex

### 2.1 进入交互模式

```bash
codex
```

作用：

- 打开 Codex 交互界面
- 进入当前目录开始协作
- 这也是“新建一个新会话”的最直接方式

适合：

- 日常写代码
- 看项目
- 修 bug
- 对话式协作

### 2.2 启动时顺带给一句任务

```bash
codex "先分析这个项目的结构，不要修改文件"
```

作用：

- 启动后直接带着第一句任务进入会话

适合：

- 你已经知道第一步想让它做什么

## 4. 最常用的启动参数

### 4.1 指定工作目录

```bash
codex --cd services/payments
```

作用：

- 让 Codex 从指定目录开始工作

适合：

- 你只想让它看一个子项目或子模块

### 4.2 增加额外可写目录

```bash
codex --cd apps/frontend --add-dir ../shared --add-dir ../backend
```

作用：

- 主工作目录之外，再允许访问额外目录

适合：

- 跨模块协作
- 前后端一起改

### 4.3 指定模型

```bash
codex --model gpt-5.4
```

作用：

- 启动时直接指定模型

适合：

- 你已经明确要用哪个模型

### 4.4 开启网页搜索

```bash
codex --search
```

作用：

- 允许 Codex 在会话里使用网页搜索能力

适合：

- 查最新信息
- 查官方文档
- 查版本更新

### 4.5 低摩擦自动执行

```bash
codex --full-auto
```

作用：

- 用更顺手的自动执行方式启动
- 等价于常见的“工作区可写 + 按需审批”组合

适合：

- 日常本地开发
- 你想减少频繁确认

## 5. 权限和沙箱相关

### 5.1 指定沙箱模式

```bash
codex --sandbox read-only
codex --sandbox workspace-write
codex --sandbox danger-full-access
```

作用：

- 控制 Codex 能读写和执行到什么程度

常见理解：

- `read-only`：只看不改，适合分析
- `workspace-write`：能改当前工作区，最常用
- `danger-full-access`：完全放开，谨慎使用

### 5.2 指定审批策略

```bash
codex --ask-for-approval untrusted
codex --ask-for-approval on-request
codex --ask-for-approval never
```

作用：

- 控制 Codex 什么时候需要问你再执行命令

常见理解：

- `untrusted`：更保守
- `on-request`：日常最常见
- `never`：不问，适合自动化

## 6. 最常用的子命令

### 6.1 非交互执行

```bash
codex exec "修复 tests 目录里失败的测试，并运行相关验证"
```

作用：

- 不进入长会话，直接跑一次任务

适合：

- 一次性任务
- 脚本化调用

### 6.2 非交互 review

```bash
codex review
```

作用：

- 直接做代码审查

适合：

- review 改动
- 检查 bug、风险、遗漏测试

### 6.3 登录管理

```bash
codex login
codex login status
codex logout
```

作用：

- 登录或退出登录
- 查看当前登录状态

### 6.4 恢复上一次会话

```bash
codex resume
codex resume --last
```

作用：

- 继续之前的会话

适合：

- 昨天没做完
- 中途切走后回来继续

### 6.5 从旧会话分叉

```bash
codex fork
codex fork --last
```

作用：

- 基于旧会话复制一条新线程

适合：

- 试另一种方案
- 不想污染原会话

### 6.6 应用最近一次改动

```bash
codex apply <TASK_ID>
```

作用：

- 把指定任务生成的 diff 应用到本地工作区

注意：

- 这里需要带 `TASK_ID`
- 不是无参数直接执行

### 6.7 管理 MCP

```bash
codex mcp
codex mcp-server
```

作用：

- 管理外部 MCP 服务
- 或把 Codex 自己作为 MCP server 启动

### 6.8 生成 shell 补全

```bash
codex completion
```

作用：

- 生成终端补全脚本

适合：

- 你想在 shell 里更快输入命令

## 7. 会话里最常用的斜杠命令

这部分是在 Codex 交互界面里输入，不是在系统终端里执行。


### 7.2 看或切权限

```text

```

### 7.5 压缩当前长会话

```text
/compact
```

### 7.6 继续旧会话

```text

```

### 7.7 分叉当前思路

```text

```

## 9. 你现在最应该先会的顺序

如果你是初学者，推荐按这个顺序掌握：

1. `codex`
2. `codex --help`
3. `codex --version`
4. `codex --cd <目录>`
5. `codex --model <模型>`
6. `codex resume`
7. `codex fork`
8. `/model`
9. `/permissions`
10. `/mention`

先别急着碰：

- `danger-full-access`
- `--ask-for-approval never`
- `mcp-server`
- `app-server`

因为这些更偏进阶用法。
