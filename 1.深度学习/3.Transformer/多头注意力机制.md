## 多头注意力机制

### 先从单头注意力说起

单头注意力的核心是：让序列中**每个位置的词**，都能动态地"关注"序列中其他位置的词，加权聚合信息。

具体来说，给定输入 $X$，通过三个投影矩阵生成三个角色：

$$Q = XW^Q, \quad K = XW^K, \quad V = XW^V$$

然后计算输出：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

三个角色的直觉理解：
- **Q（Query）**：当前词在"问什么"——我想找什么信息？
- **K（Key）**：其他词在"报名字"——我有什么信息供匹配？
- **V（Value）**：其他词实际携带的内容——匹配上了，我的值是多少？

$QK^T$ 度量每对词的相关性，softmax 归一化为概率权重，再用权重对 $V$ 加权求和，得到该位置的上下文表示。

**单头的问题**：只有一套 $W^Q, W^K, W^V$，只能从**一个视角**捕捉关系，表达能力受限。

---

### 多头注意力的核心思想

**并行跑 $h$ 个独立的注意力头**，每个头有自己独立的投影矩阵，让模型同时从多个子空间视角学习不同类型的关系：

$$\text{head}_i = \text{Attention}(XW_i^Q,\ XW_i^K,\ XW_i^V)$$

$$\text{MultiHead}(X) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h) \cdot W^O$$

**关键的维度设计**（以 $d_{model}=512, h=8$ 为例）：

| | 单头 | 多头（每个头） |
|---|---|---|
| Q/K/V 维度 | 512 | $512 / 8 = 64$ |
| 注意力矩阵 | $n \times n$ | $n \times n$（但在64维空间） |
| Concat后 | — | $8 \times 64 = 512$，恢复原维度 |
| 总参数量 | 相近 | 相近 |

每个头在**低维子空间**（64维）里独立计算注意力，Concat 拼接后维度还是 512，再经 $W^O$ 做一次输出投影。**计算量与单头相当，但表达能力强得多。**

---

### 一个直观例子

句子：**"银行倒闭了，河边的银行很漂亮"**

- **Head 1**（语法）：学到"倒闭"是动词，主语是第一个"银行"
- **Head 2**（语义消歧）：发现两个"银行"含义不同，第二个与"河边"关联更强
- **Head 3**（位置）：关注相邻词的局部关系

单头只能混合处理这些任务，多头让**每个头专注一种模式**，最后融合，信息更丰富。

---

### 单头 vs 多头：核心区别

| 对比维度 | 单头注意力 | 多头注意力 |
|---------|-----------|-----------|
| 关注视角 | 只有一种 | 多种，每头独立 |
| 捕捉关系 | 单一 | 多样（语法/语义/位置等） |
| 表达能力 | 较弱 | 更强 |
| 计算量 | 相近 | 相近（维度分摊） |
| 参数量 | 相近 | 相近 |

### 注意力权重的形态

#### 1. 单头注意力权重

假设句子有 **N 个词**（比如 4 个词：我 / 爱 / 北 / 京），单头注意力权重就是一张 **N × N 的分数表（矩阵）**：

```
        我   爱   北   京
我    [0.3, 0.2, 0.3, 0.2]
爱    [0.4, 0.1, 0.3, 0.2]
北    [0.2, 0.3, 0.4, 0.1]
京    [0.2, 0.2, 0.2, 0.4]
```

- 每 **一行** = 当前词对 **所有词（自身+其他）** 的关注度
- 整张表 = 这一层的**全部注意力权重**

**结论：** 注意力权重是一组值（矩阵），不是单个数字。

#### 2. 多头注意力权重

以 **8 个头** 为例：

- 头 1 → 算自己的 Q₁K₁ → 得到 **自己的一张 N×N 权重表**
- 头 2 → 算自己的 Q₂K₂ → 得到 **自己的另一张 N×N 权重表**
- …
- 头 8 → 同理，**自己独立一张权重表**

**重点：**
- **每个头都有自己完整的权重**
- **每个头都会看：自己 & 其他所有字**
- 只是每个头**关注的角度不一样**（有的看语法，有的看指代…）

### 极简总结（背这个）
- 注意力权重：**N×N 一组值（矩阵）**，不是单个数字
- 多头注意力：**每个头都独立算一套 N×N 权重，自己用自己的。**

### PyTorch 实现精髓

文件里已有完整实现，核心逻辑如下：

```python
# 生成 Q K V，用 .view() 拆分给各个头
Q = self.W_Q(x).view(B, T, h, self.d_k).transpose(1, 2)  # (B, h, T, d_k)
K = self.W_K(x).view(B, T, h, self.d_k).transpose(1, 2)
V = self.W_V(x).view(B, T, h, self.d_k).transpose(1, 2)

# 各头并行做缩放点积注意力
scores = (Q @ K.transpose(-2, -1)) / (self.d_k ** 0.5)  # (B, h, T, T)
attn = scores.softmax(dim=-1)
out  = attn @ V                                          # (B, h, T, d_k)

# 合并多头 + 输出投影
out = out.transpose(1, 2).contiguous().view(B, T, D)     # (B, T, D)
return self.W_O(out)
```

实现精髓在于三步：
1. **`.view(B, T, h, d_k)` + `.transpose(1,2)`**：将 $d_{model}$ 维度逻辑上拆分给 $h$ 个头，调整维度顺序让 GPU 并行计算各头
2. **`@ K.transpose(-2,-1)` + softmax + `@ V`**：$h$ 个头在各自的 64 维空间里同时做注意力，互不干扰
3. **`.transpose(1,2).view(B, T, D)`**：把 $h$ 个头的输出在最后一维拼接回 $d_{model}$，再经 $W^O$ 融合

整个过程**没有显式的 for 循环**，靠张量维度的巧妙排列，让 GPU 一次性并行处理所有头——这就是多头注意力实现的精髓。
