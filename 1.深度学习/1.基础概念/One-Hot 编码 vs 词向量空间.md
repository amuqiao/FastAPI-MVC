## One-Hot 编码 vs 词向量空间

### One-Hot 编码

词表大小假设为 10000，每个词用一个 10000 维的稀疏向量表示，只有对应位置为 1，其余全 0：

```
"猫" → [0, 0, 1, 0, 0, ..., 0]   (10000维)
"狗" → [0, 0, 0, 1, 0, ..., 0]   (10000维)
"猫" · "狗" = 0  ← 任意两词内积都是 0，没有语义关系
```

**本质问题：**
- 维度极高，极度稀疏
- 词与词之间完全正交，无法表达"猫和狗都是动物"这种语义
- 没有"距离"的概念

---

### 词向量空间（Word Embedding）

每个词被映射到一个低维稠密向量（如 128/256/768 维），语义相近的词在空间中距离相近：

```
"猫" → [0.21, -0.54, 0.87, ...]   (128维)
"狗" → [0.19, -0.51, 0.83, ...]   (128维)
"汽车"→ [0.92,  0.13, -0.44, ...]  (128维)

cos("猫","狗")   ≈ 0.92  ← 语义相近
cos("猫","汽车") ≈ 0.11  ← 语义无关
```

还能做语义运算：
```
"国王" - "男人" + "女人" ≈ "女王"
```

---

### 核心区别对比

| | One-Hot | 词向量 |
|---|---|---|
| 维度 | 词表大小（万级） | 固定低维（百级） |
| 稀疏性 | 极稀疏（只有1个1） | 稠密 |
| 语义关系 | 无（所有词等距） | 有（相似词距离近） |
| 词间运算 | 无意义 | 有意义（加减法） |
| 获取方式 | 规则生成 | 训练学习得到 |
| 存储开销 | 大 | 小 |

---

### 联系：Embedding 层就是 One-Hot 的查表操作

本质上，词向量矩阵 `E`（形状 `vocab_size × embed_dim`）的**每一行**就是对应词的向量。

```
One-Hot([0,0,1,0,...]) × E  =  E 的第 2 行  =  "猫"的词向量
```

所以 Embedding 层在计算上等价于用 One-Hot 做矩阵乘法，但实现时直接用索引查表，避免了稀疏矩阵运算的开销。

**一句话总结：One-Hot 是词的"身份证号"（只能区分），词向量是词的"DNA"（能表达相似性）。**
