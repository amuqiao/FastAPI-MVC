# GitHub 开源项目收纳清单

> **文档职责** 用于收纳我感兴趣的 GitHub 开源项目，方便统一记录、筛选和回顾。
> **适用场景** 适用于发现新项目时快速收集、整理重点项目，以及按主题分类沉淀项目。
> **阅读目标** 快速看清有哪些项目值得关注、分别适合什么场景、当前处于什么状态。
> **目标读者** 面向文档维护者本人，以及需要快速了解项目情况的协作者。

## 1. 简短维护规则

- 新项目先放入「收集箱」
- 值得继续跟进的项目移入「重点关注区」
- 稳定沉淀的项目放入「分类项目库」
- 表格字段保持统一，不随意新增列
- 同一个项目只保留一条主记录，避免重复
