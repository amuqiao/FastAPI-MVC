# Codebase Project Analysis

## Purpose

这个 skill 用于分析本地代码仓库，帮助用户先快速建立整体认知，再逐步深入到架构层或模块层。

它负责：
- 澄清分析目标
- 选择分析深度
- 提炼项目定位、技术栈、主模块和主链路
- 识别关键设计决策、数据模型、扩展点和耦合风险
- 在有帮助时配合 Mermaid 图辅助说明

它不负责单纯作图，也不代替技术笔记类 Prompt。

## Default Mode

默认先澄清需求，再执行 `scan`，先给简洁结论，不直接输出完整长文。

默认策略：
- 只抓主链路和主模块
- 只要图能明显降低理解成本，就默认配图
- 后续根据用户需求升级到 `architecture` 或 `module`

## Clarification First

开始分析前，默认澄清以下 4 个问题：

1. 这次更想看哪一层：整体速览、架构设计，还是某个模块？
2. 默认先做 `scan` 可以吗？
3. 有没有特别关注的模块、入口流程或目录？
4. 这次主要是为了看懂项目、看懂链路、看懂设计决策，还是为后续修改做准备？

如果用户没有补充信息，按以下默认值处理：
- 深度：`scan`
- 范围：主模块 + 主链路
- 输出：简洁结论 + 1 到 2 张图

## Depth Modes

### scan

用于快速建立全局认知。默认输出：
- 项目定位
- 技术栈
- 主模块
- 最关键执行链路
- 2 到 3 个关键设计决策
- 总共 1 到 2 张图

这里的 1 到 2 张图指总图数，不是每个栏目各配 1 到 2 张。

### architecture

用于深入理解系统结构和关键设计。默认输出：
- 项目定位
- 技术栈全景
- 主模块与职责
- 模块依赖关系
- 核心执行链路
- 关键设计决策
- 数据模型
- 扩展点机制
- 风险或高耦合区域
- 总共 2 到 4 张图

### module

用于聚焦单个模块。默认输出：
- 模块职责边界
- 入口函数或触发条件
- 模块内部主链路
- 依赖方向
- 核心数据结构
- 错误处理与边界情况
- 最小改动路径
- 总共 1 到 3 张图

## Output Contract

默认先给简洁结论，再按需展开。推荐顺序：

1. 分析范围与深度说明
2. 简洁结论
3. 必要的 Mermaid 图
4. 图解说明
5. 下一步可继续深入的方向

除非用户明确要求完整分析文档，否则不要在第一轮铺开全部细节。

## Mermaid Collaboration

当图能明显降低理解成本时，调用 Mermaid 相关规范进行表达。

图型选择原则：
- 整体架构、同步流程：优先 A
- 数据模型、类关系：优先 B
- 异步链路、状态变化：优先 C
- 规划、时间线、主题展开：优先 D

图的具体规范、语法边界和图解说明由 `mermaid-analysis-visualizer` 负责。

## Analysis Constraints

- 默认只抓主模块和主链路，不追求第一轮穷尽覆盖
- 优先解释为什么这样设计，而不是只描述是什么
- 不把猜测写成事实
- 用户未指定时，优先选择最能建立整体认知的角度
- 项目很大时，先给主线，再提示可继续深挖的方向

## Self-Check

输出前检查：
- 当前深度是否匹配用户目标
- 是否抓住了主模块和主链路
- 是否过早展开过多细节
- Mermaid 图是否真的带来理解收益
- 是否明确区分了事实、推断和待核验项
- 是否给出了清楚的下一步深入建议
